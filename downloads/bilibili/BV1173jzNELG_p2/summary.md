## 概述
本视频是吴恩达Prompt Engineering教程的第二部分，主要讲解提示词编写的两个核心原则：一是编写清晰具体的指令，二是让模型有思考时间。视频通过Jupyter Notebook演示了多个实用的提示工程策略，并介绍了模型的局限性及如何避免幻觉问题。

## 关键要点
1. **两个核心原则**：清晰具体指令 + 给予模型思考时间
2. **第一原则的四个策略**：使用分隔符、结构化输出、条件检查、少样本提示
3. **第二原则的两个策略**：明确任务步骤、指导模型自行推导解决方案
4. **模型局限性**：存在"幻觉"问题，可能编造看似合理但实际虚假的内容
5. **减少幻觉的方法**：先从文本中提取相关引文，再基于引文回答问题

## 详细内容

### 课程准备
- 安装OpenAI Python库：`pip install openai`
- 设置API密钥（可设为环境变量）
- 使用GPT-3.5 Turbo模型和聊天完成接口
- 定义辅助函数用于获取提示词生成结果

### 第一原则：编写清晰具体的指令

**策略一：使用分隔符**
- 使用三引号、XML标签等明确标示输入文本的不同部分
- 有效避免"提示注入"问题（即用户输入与模型指令冲突）

**策略二：要求结构化输出**
- 要求模型输出HTML或JSON等格式
- 便于程序直接读取和处理

**策略三：检查条件是否满足**
- 让模型先验证假设条件
- 处理边界情况，避免错误结果

**策略四：少样本提示**
- 提供成功案例示例
- 让模型学习所需的任务风格和格式

### 第二原则：让模型有思考时间

**策略一：明确任务所需步骤**
- 列出具体步骤要求
- 指定明确的输出格式（视频中使用尖括号作为分隔符示例）

**策略二：指导模型自行推导解决方案**
- 要求模型先独立解题，再与给定答案对比
- 避免模型快速浏览导致推理错误
- 视频示例：让学生解答与正确答案进行对比验证

### 模型的局限性

**幻觉（Hallucinations）**
- 模型可能编造看似合理但实际不存在的信息
- 危险在于描述非常真实，难以辨别

**减少幻觉的方法**
- 基于文本生成答案时，先提取相关引文
- 要求模型使用引文回答问题，并可追溯来源

## 重要概念
- **提示工程（Prompt Engineering）**：优化提示词以获得更好结果的实践
- **提示注入（Prompt Injection）**：用户输入与模型指令冲突的安全问题
- **少样本提示（Few-shot Prompting）**：通过示例引导模型完成任务
- **幻觉（Hallucination）**：模型生成虚构信息的现象
- **结构化输出**：要求模型以特定格式（如JSON）输出结果
- **推理链条**：让模型分步骤思考以提高准确性
