概述

本视频对Claude Code的18个命令进行实用评级，从"顶级"到"拉"五个等级，帮助用户了解哪些命令是真正提升效率的神器，哪些是几乎无用的占位符。作为Cloud重度用户，作者分享了自己日常使用中最常用的命令评价。

关键要点 (按评分排序)

顶级 (6个)

  1. hooks - 自动化工作流配置，一次配置爽一年
  2. MCP - 连接外部工具和数据库，AI变身超级助理
  3. INIT - 初始化项目生成CLAUDE.md配置文件
  4. BTW - 补充背景信息不打断对话，高频实用
  5. loop - 循环任务调度，定时自动执行监控

夯/夯爆了 (6个)

  1. compact - 压缩对话历史省token，长对话救命稻草
  2. agent - 启动子代理多文件并行操作
  3. clear - 清空上下文重新开始
  4. plan - 规划模式先写方案再动手
  5. skills - 管理自定义技能，性价比之王

人上人 (3个)

  1. DEFE (diff) - 查看AI动了哪些文件
  2. context - 检查云端记住了什么
  3. insights - 代码洞察分析，接手史山项目必备

NPC (5个)

  1. help - 查帮助文档，新手村NPC
  2. /copy - 复制对话到剪贴板
  3. cost - 查看token消耗明细
  4. Dr. (diagnose) - 环境诊断

拉 (1个)

  1. exit - 退出会话，作者认为完全没必要

详细内容

顶级评价命令

hooks - 配置钩子 (顶级) 在特定事件时自动执行命令，如每次提交前跑测试。配置完成后开发流程丝滑，但配置过程复杂，各种配置文件让人头大。自动化程度直接拉满。

MCP - 管理MCP服务器 (顶级) 让Claude连接外部工具、数据库和API。搞明白后Claude从普通AI变身超级助理，但刚接触时会被服务器配置、协议对接劝退。主要问题是过于花费token。

INIT - 初始化项目 (顶级) 生成CLAUDE.md配置文件，让Claude知道项目规矩、技术栈。第一次使用Claude Code必跑，每次打开新窗口都要用，不然Claude会开始捣乱。

BTW (顺便说一下) (顶级) 在不打断对话的前提下补充背景信息，下轮对话自动应用。如指定使用TypeScript就不用重复。适合补充遗漏信息或修正错误，高频实用指令。

loop - 循环任务调度 (顶级) 定时自动执行监控、部署状态、定期检查等任务。相当于给Claude装自动巡航系统。缺点是任务配错会被批量犯错。

夯/夯爆了评价命令

compact - 压缩对话历史 (夯) 压缩对话历史省token，但关键信息全保留。当对话太长 Claude开始失忆但又不想丢上下文时，是救命稻草。

agent - 启动子代理 (夯) 让Claude分身干活，一个查代码一个写测试。大项目重构、多文件并行操作效率起飞，但需要脑力做好任务拆分。

clear - 清空上下文 (夯) 当Claude开始胡说八道、陷入死循环或想换话题时使用。用完感觉像换了一个新的Code，混乱逻辑瞬间归零。

plan - 规划模式 (夯) 让Claude先写方案再动手。复杂任务不确定怎么实现时，先画蓝图再搬砖。真正懂工程的人用这个避免返工。

skills - 管理自定义技能 (夯爆了) 给Claude装各种扩展能力，重复性知识性工作自动化。自己写skill让它记住，相当于给Claude装插件，上下文占用少，性价比之王。

人上人评价命令

DEFE (diff) - 查看改动 (人上人) 查看AI每轮对话动了哪些文件，按左右键切换所有改动和每轮改动，按上下键浏览不同文件。程序员看小白或markdown修改，防止AI误解意思。

context - 查看上下文 (人上人) 检查当前上下文里Claude记住了哪些文件。加了太多文件Claude会选择性失忆，用这个偷看它的记事本，看它有没有在认真听。

insights - 代码洞察分析 (人上人) 帮助理解代码结构和潜在问题。接手史山项目或想了解代码在干嘛，能快速理清架构逻辑，找出性能瓶颈和设计缺陷。考古学家洛阳铲，接手无文档老项目必备。

NPC评价命令

help - 查帮助文档 (NPC) 第一次用或忘了命令叫什么时翻一翻。还算全面，基本所有斜杠命令都能找到。新手村聊两句，后面谁还找它，存在感比空气还稀薄。

/copy - 复制对话 (NPC) 把对话内容复制到剪贴板，写文档或跟同事分享时用。能复制整个对话格式化版本，但直接鼠标选中复制也行，必要性存疑。

cost - 查看消耗 (NPC) 看当前对话花了多少钱，token消耗明细。老板问AI花了多少时或自己好奇时用。别每次看，每周看一次心理承受范围内。

Dr. - 诊断环境 (NPC) 检查Claude Code环境配置有没有问题，出bug或连不上API时先跑这个。90%问题能找到原因，就像电脑重启。但诊断报告还算详细，不至于抓瞎。

拉评价命令

exit - 退出 (拉) 退出Claude Code结束当前会话，比直接关窗口优雅一点。作者认为有强迫症的程序员总觉得要有个正式结束仪式，但真的需要这个命令吗？直接CTRL+C就行，严重怀疑产品经理凑KPI。
