## 概述
本课主要介绍如何使用Google ADK（Agent Development Kit）构建多智能体系统。课程从环境配置开始，逐步讲解如何导入必要库、设置LLM模型、连接Neo4j图数据库、创建自定义工具、定义智能体以及设置执行环境，最终实现一个简单的Hello World智能体交互示例。

## 关键要点
1. **环境配置**：导入os、标准库、Google ADK核心组件（Agent、Runner）、轻量级LLM库及记忆服务
2. **模型设置**：使用OpenAI的GPT-4o模型，通过轻量级LLM库进行API调用
3. **Neo4j集成**：使用new4j库连接图数据库，创建辅助函数将结果转换为Google ADK可读格式
4. **工具创建**：定义say_hello工具函数，包含规范的文档字符串和参数说明，返回符合Google ADK规范的字典格式
5. **智能体定义**：创建Agent实例，包含名称、模型、描述、指令和可用工具
6. **执行环境**：设置Runner管理事件循环，配置SessionService进行会话管理和内存存储
7. **运行机制**：通过事件循环异步执行智能体，支持详细输出模式用于调试

## 详细内容

### 1. 导入模块
- 标准库：os
- Google ADK：Agent、Runner等核心类
- LLM通信：轻量级封装库
- 图数据库：new4j for ADK
- 类型定义：typing相关

### 2. 模型配置
- 使用GPT-4o模型
- 通过轻量级LLM库调用OpenAI API
- 测试连接确认系统正常运行

### 3. Neo4j集成
- 创建Graph4jWrapper类封装驱动程序
- 提供success/error辅助函数返回规范化结果
- 实现查询结果的序列化转换

### 4. 工具定义
- 函数形式定义工具
- 文档字符串是关键，供LLM理解工具功能
- 使用查询参数避免注入攻击
- 示例：say_hello工具返回个性化问候

### 5. 智能体配置
- **名称**：hello_agent_v_one
- **模型**：已配置的LLM
- **描述**：让其他智能体理解该智能体的用途
- **指令**：指导智能体何时及如何使用工具
- **工具列表**：传入可用工具函数数组

### 6. 执行环境
- **Runner**：管理事件循环，协调LLM调用
- **SessionService**：提供内存存储和会话管理
- **事件循环**：处理异步事件，支持详细输出调试

### 7. 封装工具
- 创建AgentCaller类封装调用逻辑
- 提供make_agent_caller工厂方法简化实例创建

## 重要概念
- **Google ADK**：Google的智能体开发工具包
- **Agent**：具有推理和工具使用能力的智能体
- **Tool**：智能体与外部环境交互的接口
- **Runner**：管理智能体执行环境的组件
- **SessionService**：提供记忆和会话管理服务
- **事件循环**：异步处理智能体消息和响应的机制
- **查询参数**：防止注入攻击的安全实践
