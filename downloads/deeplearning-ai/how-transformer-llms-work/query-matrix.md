**不是固定的。** 无论是在模型**训练阶段**还是在**推理（使用）阶段**，Query 矩阵（以及 Key、Value 矩阵）都不是一成不变的固定死值。

我们要区分两个概念：**“权重矩阵”**（参数）和“计算出来的 Query 矩阵”（动态数据）。

为了让你彻底明白它是怎么变动的，我们可以从以下两个维度来看：

---

### 1. 随着“输入内容”的不同而动态变化（推理阶段）

在模型训练好之后，它的参数（权重）就冻结了。但面对不同的输入文本，最终参与注意力计算的 Query 矩阵是**实时计算、动态生成**的。

在数学上，Query 矩阵（通常记为 $Q$）是由输入句子的词向量矩阵（$X$）乘以一个权重矩阵（$W_Q$）得到的：


$$Q = X \times W_Q$$

* **$W_Q$ 是固定的**：模型训练好后，这个映射规律就确定了。
* **$X$ 是多变的**：你输入的句子不同，或者同一个词在句子中的位置不同，$X$ 就完全不同。
* **结果 $Q$ 是动态的**：因为输入 $X$ 在变，所以相乘得到的 $Q$ 每次都在变。

> **举个例子**：
> 假设 $W_Q$ 像是一个“翻译官”，它的能力（参数）是固定的。
> * 如果你给它输入 $X_1 =$ “苹果手机”，它动态产出的 $Q_1$ 就会去找“科技、电子”相关的词。
> * 如果你给它输入 $X_2 =$ “青苹果”，它动态产出的 $Q_2$ 就会去找“水果、味道”相关的词。
> 
> 

---

### 2. 随着“训练过程”的推进而不断更新（训练阶段）

如果你指的是那个决定映射规则的权重矩阵 $W_Q$，它在**模型训练阶段**也是不固定的。

* **初始化**：在最开始， $W_Q$ 里面全都是随机生成的乱码数字。这时候模型根本不知道该去“查询”什么。
* **反向传播（Learning）**：随着海量文本的输入，模型发现自己猜错词了（损失函数很高）。通过**反向传播算法**，梯度会流回 $W_Q$，不断微调里面的每一个数值。
* **最终定型**：直到模型训练结束，$W_Q$ 才会被“冻结”下来，形成最终的完备形态。

---

### 3. 在“多头注意力”中，各头的 Query 矩阵各不相同

即使在同一个句子中，不同的**注意力头（Attention Head）**拥有的权重矩阵（$W_{Q1}, W_{Q2}, W_{Q3} \dots$）也是**互不相同**的。

每个头都有自己独立的进化方向：

* 头 1 的 $W_{Q1}$ 被训练得对“动词”敏感，它生成的 Query 总是试图去寻找句子的谓语。
* 头 2 的 $W_{Q2}$ 被训练得对“代词”敏感，它生成的 Query 总是试图去寻找“他/她/它”指代的是谁。

### 总结

* **作为参数的权重矩阵 $W_Q$**：训练时在变，训练完后固定，不同注意力头之间互不相同。
* **实际参与运算的 Query 矩阵 $Q$**：它是输入文本与 $W_Q$ 相乘的产物，**只要输入的句子在变，它就时刻在动态变化**。
