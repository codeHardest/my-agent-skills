## 内容概述
本节课介绍了如何在终端之外使用 Claude Code，包括通过 GitHub 集成在 Pull Request 中进行代码审查、在 Issue 中修复问题，以及通过 Claude Code Hooks 在工具执行前后的生命周期中注入自定义代码。课程通过实际演示展示了完整的 GitHub 工作流程和 Hooks 配置方法。

## 核心要点
1. 使用 `/install-GitHub-app` 命令安装 Claude Code 的 GitHub 应用集成
2. 通过工作流文件实现自动代码审查和 Issue 标记功能
3. Claude 可以自动分析 Pull Request 中的代码质量、安全问题和最佳实践
4. 可以在 GitHub Issue 中 @mention Claude 来自动生成修复代码的 Pull Request
5. Claude Code 生成的代码会经过另一个 Claude 实例进行自我审查
6. Hooks 功能允许在 Claude Code 操作的生命周期中注入自定义代码
7. 支持多种 Hook 事件类型：PreToolUse、PostToolUse、onNotification、onPrompt、onStop、PreSubagentConclusion
8. 可以使用 matcher 配置让 Hook 只在特定工具被调用时触发
9. Hooks 配置存储在 `.claude/settings.local.json` 文件中
10. 工作trees 可以通过 Claude Code 清理并同步到 GitHub
11. GitHub Actions 会自动运行 Claude 进行代码审查
12. 可以在代码审查配置文件中自定义审查提示词
13. Hooks 可以用于运行测试、代码检查或阻止特定工具执行
14. 下节课将介绍在 Jupyter Notebooks 中使用 Claude Code

## 详细内容（保留所有原始信息）

### 1. 清理工作trees并推送到GitHub
课程开始时回顾了之前创建的工作trees操作，需要清理这些工作trees。使用 resume flag 恢复之前的会话，然后让 Claude 执行以下操作：
- 移除 .trees 文件夹及其下的所有 worktrees
- 删除对应的分支
- 将合并后的代码推送到 GitHub

Claude Code 会依次执行 git 命令来移除目录、删除分支，最后确认执行 `git push origin main` 将代码推送至 GitHub。

### 2. 安装GitHub集成
完成代码推送后，开始安装 Claude Code 的 GitHub 集成功能：
- 使用 `/install-GitHub-app` 命令开始安装
- 可能需要额外的命令行界面认证，按照提示完成设置
- 安装 GitHub App 到当前仓库
- 安装过程会打开浏览器完成授权

**GitHub集成的核心能力**：
- 在 Pull Request 中响应反馈、修复错误、修改代码
- 在 Issue 中进行交流和问题修复
- 该功能基于 Claude Code SDK 构建，允许在终端界面之外使用 Claude Code

### 3. 配置工作流(Workflows)
返回终端后，需要指定要安装的工作流：
- **标记功能工作流**：允许在 Issue 中 @mention Claude
- **代码审查工作流**：自动在 Pull Request 中进行代码审查

安装步骤：
- 创建长期令牌 (long-lived token)
- 完成与 Claude 的授权和认证
- 配置完成后自动跳回 GitHub 界面

### 4. 创建Pull Request与自动代码审查
安装工作流后，会自动打开一个 Pull Request，包含：
- Claude 操作所需的 YAML 配置文件
- 代码审查的配置文件

**代码审查功能特性**：
- 预设了合理的默认配置
- 可以按作者过滤
- 可以指定运行环境
- 可以修改审查提示词
- 配置文件受 git 跟踪，可以持续编辑

合并 Pull Request 后，Claude GitHub Action 会自动启动，开始进行代码审查：
- 读取和分析文件
- 检查代码质量
- 识别安全问题
- 提供详细的评估报告

Claude 作为团队的新成员，能够检查开发人员的工作，在人工可能遗漏的地方进行额外检查。

### 5. 使用Claude修复GitHub Issue
演示场景：团队成员添加了一个新 Issue，内容是应用程序添加了新 Header，希望恢复到之前的设计，但保留主题切换功能。

**具体要求**：
- 恢复到旧的 Header 外观
- 保留主题切换 (toggle theme) 功能
- 移除 "Course Materials Assistant" Header
- 移除 "around ask questions" 子标题
- 移除子标题下方的水平分隔线

**修复流程**：
1. 在 GitHub 上创建 Issue，描述需要修复的问题
2. 在 Issue 中 @mention Claude："Claude, can you fix this for me?"
3. Claude Code 开始分析问题，查看底层 job 运行情况
4. Claude 分析代码结构，制定修复计划
5. 修复完成后，Claude 会：
   - 测试更改
   - 提交代码
   - 创建 Pull Request 修复该 Issue

**Claude的自我审查**：
- Claude 生成的代码还会经过另一个 Claude 实例进行审查
- 这提供了双重检查机制，增强代码质量
- 审查完成后可以合并 Pull Request

**最终验证**：
- 回到终端，在 VS Code 中拉取更改
- 确认前端效果：Header 被移除，水平分隔线也被移除
- 用户可以选择恢复水平分隔线或调整位置

### 6. Claude Code Hooks 功能介绍
最后介绍了一个最近发布的功能——Hooks（钩子）。

**Hooks的核心概念**：
- 在 Claude Code 的不同操作阶段注入自定义代码
- 可以在工具执行前、执行后或特定事件发生时触发
- 类似于传统的 Web 开发 Hooks 概念

**可用的Hook事件类型**：
| 事件类型 | 说明 |
|---------|------|
| PreToolUse | 工具执行前触发，可阻止工具执行 |
| PostToolUse | 工具执行后触发 |
| onNotification | 通知发送时触发 |
| onPrompt | 用户提交提示词时触发 |
| onStop | 任务停止时触发 |
| PreSubagentConclusion | 子代理完成响应前触发 |

**使用注意事项**：
- 如果执行任意 shell 命令，必须非常谨慎
- Hooks 配置存放在 `.claude/settings.local.json` 文件中
- 可以使用 `/hooks` 命令管理工具事件配置

### 7. Hooks 配置示例
演示了一个简单的 PostToolUse Hook 示例：

**场景**：在 Read 或 Grep 工具执行后，让电脑朗读 "All done!"

**配置步骤**：
1. 添加 matcher（匹配器）
2. 指定要匹配的工具：Read 和 Grep
3. 配置要执行的命令：使用 Mac 的 `say` 命令

**配置代码示例**：
```json
{
  "permissions": { ... },
  "hooks": {
    "PostToolUse": {
      "matcher": {
        "tools": ["Read", "Grep"]
      },
      "command": "say 'All done!'"
    }
  }
}
```

**测试验证**：
- 重启 Claude Code
- 执行读取文件的操作
- 工具执行完成后，系统会朗读 "All done!"

**实际应用场景**：
- 运行测试
- 执行代码检查 (linter)
- 阻止特定工具使用
- 在特定事件发生时让 Claude 自我审查
