## 内容概述
本课程展示了如何使用Claude Code重构混乱的Jupyter笔记本，并将其转换为专业的Streamlit仪表板。讲师从电子商务数据的分析笔记本开始，演示了代码重构、分离关注点、创建辅助模块、优化可视化效果，最终生成可部署的Web仪表板应用程序。

## 核心要点
1. Claude Code具有读写和编辑Jupyter笔记本的工具，可以使用`read notebook`命令来读取笔记本内容
2. 重构笔记本时应创建独立的Python文件处理数据加载、清洗和业务指标计算
3. 分离业务逻辑与展示层有利于代码测试和理解维护
4. 重构过程中可能遇到KeyError等错误，可通过Claude Code辅助调试修复
5. Streamlit是Python库，可将笔记本无缝转换为Web仪表板
6. Streamlit仪表板应包含标题/过滤器头部、KPI卡片（收入、增长、平均订单价值、订单总量）
7. 可使用Plotly等可视化库创建交互式图表
8. 仪表板应支持按年份和月份筛选数据，默认值设置为2023年较为合理
9. 自定义CSS可用于美化和布局仪表板界面
10. 配置文件应包含依赖列表（requirements.txt）和使用说明（README）
11. 迭代式开发是使用Claude Code的关键，可逐步完善功能
12. 提示工程技巧：可直接询问Claude获取最佳提示词示例
13. 生成的辅助模块采用面向对象方法，比直接在notebook中编写更清晰
14. 生成的代码包含数据加载类、指标计算类和可视化函数
15. 仪表板可按州显示地理收入可视化，支持悬停查看详细信息

## 详细内容（保留所有原始信息）

### 问题背景：混乱的Jupyter笔记本

课程开始时，讲师展示了一个包含电子商务数据的Jupyter笔记本。该笔记本存在以下问题：
- 数据来自ecommerce_data文件夹中的多个CSV文件，使用pandas读取
- 虽然能获取所需答案，但数据组织不够规范
- 统计信息格式和显示效果不理想
- 存在warnings警告信息
- 涉及多个CSV和数据框的合并操作，计算过程繁琐
- 可视化展示不够直观美观
- 仅针对2023年计算高阶指标如平均订单价值，若需2024年数据则需重复工作
- 销售分州统计等操作虽然能生成可视化，但整体效果仍有提升空间

### 重构策略与实现过程

讲师展示了如何利用Claude Code进行笔记本重构：

**第一步：准备重构提示词**
- 创建新的markdown文件记录重构需求
- 在提示中使用文件引用符号（@）引用笔记本和相关文件
- 可直接询问Claude获取最佳提示示例
- 重构需求包括：
  - 代码质量改进
  - 创建单独的Python文件用于数据加载和处理
  - 创建Python文件用于计算业务指标
  - 改进可视化效果
  - 增加数据配置灵活性
  - 生成requirements.txt和README文件

**第二步：执行重构任务**
- Claude Code使用`read notebook`命令替代普通read命令
- 分析笔记本结构后创建重构后的notebook
- 同时生成辅助Python文件：
  - 数据加载处理模块（面向对象设计）
  - 业务指标计算模块（包含数据验证和可视化功能）
- 生成requirements.txt和README文件
- 执行过程中可随时检查生成的文件内容

**第三步：调试与修复**
- 运行重构后的notebook时发现KeyError
- 将错误信息和相关代码提供给Claude Code进行诊断修复
- 修复后继续运行，确认问题解决

### 重构后的改进效果

重构后的notebook呈现显著改进：
- 视觉效果更加美观
- 提供数据配置选项，可指定年份、对比数据、月份筛选
- 使用常量贯穿整个notebook
- 加载自定义模块（类）进行数据处理
- 业务指标展示更清晰易读
- 客户满意度分析展示更加友好
- 收入分析图表更加清晰美观
- 类别性能分析工具更加完善
- 地理性能可视化支持按州显示，可悬停查看详细信息
- 配送分析等额外内容展示正常

### 从笔记本迁移到Streamlit仪表板

**准备工作：创建转换提示**
- 创建convert-to-dashboard.md文件
- 明确仪表板布局要求：
  - 头部：标题和过滤器
  - KPI卡片：收入、增长、平均订单价值、总订单量
  - 图表：收入趋势、类别收入、州收入、满意度柱状图
  - 底部行：两张卡片展示特定数据

**实现过程**
- 使用Streamlit创建Web界面
- 导入必要库：Streamlit、pandas、Plotly
- 添加自定义CSS样式
- 设置页面配置（使用st对象）
- 更新requirements.txt
- 修改现有README文件说明仪表板功能

**启动服务**
- 使用streamlit run dashboard.py启动服务
- 如缺少依赖，使用pip install安装所需包
- 浏览器中查看仪表板效果

### 迭代优化

初始仪表板存在以下问题：
- 部分卡片为空且冗余
- 默认年份为2024，但缺少年度对比数据
- 月度增长数据因2024数据不完整而显示混乱

优化措施：
- 设置默认年份为2023
- 添加月份筛选器
- 移除空卡片
- 更新收入图表以处理月份筛选情况
- 优化CSS使图表容器更简洁
