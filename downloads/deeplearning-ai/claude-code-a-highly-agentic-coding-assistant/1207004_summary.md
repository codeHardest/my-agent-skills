## 内容概述
本视频展示了如何使用Claude Code为应用程序添加功能特性，包括前端界面优化（添加源引用链接、"新对话"按钮）和后端工具开发（课程详情工具）。教程重点演示了文件引用、计划模式、自动接受编辑、截图辅助等核心功能，以及如何通过MCP服务器（如Playwright）实现自动化测试和验证。

## 核心要点
1. **正确引用文件**：Claude Code的效果取决于提供的上下文质量，提前知道需要修改的文件并引用它们，可以让工具更加高效

3. **计划模式（Plan Mode）**：对于较大改动，建议使用计划模式，让Claude先制定详细方案，用户批准后再执行
4. **激活计划模式**：按Shift+Tab两次可以激活计划模式
5. **灵活的审批机制**：可以手动批准每次修改、自动接受修改、或要求Claude修改计划
6. **自动接受编辑**：按Shift+Tab一次可以开启自动接受编辑功能，适合信任度建立后使用
7. **截图辅助功能**：Claude可以分析截图来理解UI问题，这是一个非常强大和常用的功能
8. **MCP服务器**：Model Context Protocol允许Claude Code连接外部工具和服务进行扩展
9. **Playwright MCP**：通过`npx @playwright/mcp@latest`命令添加Playwright MCP服务器
10. **CLAUDE.md配置**：可以将偏好设置写入CLAUDE.md文件供所有开发者参考
11. **CLAUDE.local.md本地配置**：本地配置使用.gitignore忽略，适合个人偏好设置
12. **清空对话历史**：开始新功能时清空对话历史可以避免混淆，充分利用上下文窗口
13. **Esc键中断**：按Esc键可以中断Claude的操作并重新引导方向
14. **后端工具开发**：需要同时修改Python代码、系统提示词和RAG系统

## 详细内容（保留所有原始信息）

### 一、添加源引用链接功能

在现有应用中，当用户请求课程大纲时，可以获得详细的课程信息和参考来源。然而，这些参考来源仅以文本形式显示，无法点击访问。用户希望能够点击这些来源链接，直接跳转到原始资料。

实现这个功能需要同时修改前端和后端代码，确保链接能够正确渲染显示。前端需要正确显示可点击的链接样式，后端需要提供正确的数据格式。

### 二、计划模式的工作流程

#### 2.1 激活计划模式
- 按Shift+Tab两次激活计划模式
- 系统会显示"plan mode is on"

#### 2.2 提交需求
- 用户提供详细的提示词
- 可以引用需要修改的正确文件
- 也可以引用需要修改的其他相关文件

#### 2.3 Claude分析制定计划
- Claude会读取它认为必要的文件
- 从前端到后端追踪理解需要实现的功能
- 生成详细的实施计划

#### 2.4 用户审批
- 可以接受计划并继续
- 可以手动批准每次修改
- 可以要求Claude修改计划
- 可以按Escape键中断并引导不同方向

#### 2.5 执行实施
- 使用自动接受编辑功能可以减少重复确认
- 按Shift+Tab一次可以开启自动接受
- Claude会修改必要的文件
- 完成后可以测试功能是否正常

### 三、"新对话"按钮的实现

#### 3.1 需求背景
用户希望添加一个按钮来重置对话，无需刷新页面即可开始新的对话。

#### 3.2 实现步骤
1. 激活计划模式
2. 提供详细的需求说明：
   - 添加"新对话"按钮
   - 点击时清空聊天窗口的对话
   - 开始新会话
   - 处理必要的清理工作
3. Claude会制定完整计划并实施

#### 3.3 功能实现内容
- 添加JavaScript代码处理点击事件
- 实现清空输入框和聚焦功能
- 添加样式以匹配现有界面设计
- 测试新功能是否正常工作

#### 3.4 服务器启动配置
- 可以将"不要运行服务器"等偏好写入CLAUDE.md
- 本地偏好可以写入CLAUDE.local.md（会被.gitignore忽略）
- 这样可以方便自己使用，同时不影响其他开发者

### 四、截图辅助功能

#### 4.1 基本用法
- 截取界面的截图
- 将截图粘贴到Claude Code中
- 描述问题所在，如"这些链接难以阅读，请美化界面"

#### 4.2 Claude分析能力
- Claude可以分析截图中的视觉问题
- 自动定位需要修改的前端代码位置
- 生成相应的修改方案

#### 4.3 迭代改进
- 修改后可以再次截图验证效果
- 可以继续提出修改要求
- 直到界面达到满意效果

### 五、MCP服务器集成

#### 5.1 什么是MCP
Model Context Protocol（MCP）允许Claude Code获得额外的功能扩展，连接外部数据源和系统。

#### 5.2 添加Playwright MCP服务器
退出Claude Code后，使用以下命令添加：
```bash
claude mcp add playwright npx @playwright/mcp@latest
```

#### 5.3 Playwright提供的工具
- evaluate_javascript：执行JavaScript代码
- upload_file：上传文件
- press_key：按键操作
- navigate：导航操作
- go_back：返回操作
- visit：访问页面
- screenshot：截图

#### 5.4 自动化测试流程
1. 使用Playwright MCP自动打开浏览器访问页面
2. 拍摄截图分析当前界面
3. 识别问题并自动修复
4. 再次截图验证修改效果
5. 继续迭代直到满意

#### 5.5 实际应用示例
用户希望"新对话"按钮的样式与其他链接保持一致，要求：
- 左对齐
- 移除边框
- 样式与"Courses"和"Try Asking"下方的链接一致

Claude使用Playwright MCP自动：
1. 打开新标签页访问页面
2. 拍摄截图分析
3. 识别问题并进行修改
4. 再次截图验证效果
5. 修复后发现多余的加号图标，手动移除

#### 6.1 需求背景

- 课程编号
- 课程标题
- 课程描述

#### 6.2 实现步骤
1. 清空对话历史开始新会话
2. 使用计划模式规划实现
3. 修改search_tools.py文件
4. 更新系统提示词

#### 6.3 CLAUDE.md的作用
由于之前已经创建了CLAUDE.md文件，Claude Code能够更快地理解项目架构，更高效地完成开发任务。

#### 6.4 最终效果
用户可以询问课程详情，获得每个课程章节的详细信息，以及相关链接。
