## 详细内容（保留所有原始信息）

### 一、Skills作为开放标准

本课承接上一课关于Claude中创建Skills的内容，进一步深入探讨Skills的本质及其背后的开放标准。与Model Context Protocol类似，Skills本身是一个开放标准，可用于许多不同的AI应用程序。虽然Skills最初由Anthropic创建，但现在Skills已成为一个开放标准，具有特定规范，被用于许多不同平台，包括Codex、Gemini CLI、Claude Code、Open Code等。

这一开放标准的特性意味着，用户可以在一个环境中创建Skills，然后在许多不同环境中使用、分享和扩展它们，极大地提高了Skills的实用性和可移植性。

### 二、Skills的技术实现

#### 文件系统架构

当使用Claude AI或Claude Desktop等工具构建AI应用程序时，为了使用特定Skills，需要利用某种文件系统。在该文件系统中，加载包含SKILL.md文件的文件夹，以及可以引用的子文件夹或文件。这种架构设计使得Skills具有清晰的结构和良好的组织性。

#### 超越文本文件的扩展性

Skills本身不仅能包含其他Markdown文档，还能包含可执行脚本。例如，处理PDF文档的Skills需要将PDF转换为图像、从表单字段提取信息、甚至用注释填充PDF表单——这都需要执行代码。而这些需要执行的代码可以从SKILL.md文件中引用。

这意味着Skills不仅仅是引用其他文本文件的文本文件，而是可以引用脚本、说明脚本做什么、以及何时需要执行的完整系统。

#### 资产支持

Skills还可以包含图标、图片和其他资产，这对于创建自定义样式和品牌至关重要。企业可以建立具有统一视觉识别系统的Skills，确保输出内容符合品牌形象。

### 三、引入Skills的核心价值

#### 领域专业知识

Skills真正发挥作用的场景是Claude可能不完全了解你或你的公司如何运作的领域。例如设计新闻通讯、创建品牌指南等——Claude对此有大致的了解，但不了解你的公司或团队的具体运作方式。

虽然Claude可以进行数据分析、法律审查等工作，但关键问题是它是否按照你或你的团队或公司期望的方式执行这些任务。Skills提供了程序性知识和用户特定上下文，可以在需要时加载。

#### 从单用途代理到模块化架构

过去构建代理的思路主要围绕单一目的的代理：编码、研究、金融、营销等。这些特定领域的代理有执行任务所需的特定工具集和上下文。但随着构建越来越多这种单用途代理，人们开始意识到它们在底层真正需要的只是简单的脚手架——bash和文件系统等底层工具，用于查找、编辑、修改、执行和执行任何必要的任务。

这些更简单的代理更容易评估、理解和扩展，但它们缺乏的是执行任务的可靠底层上下文和领域专业知识。这些上下文可以通过Skills和Model Context Protocol提供，而领域专业知识正是Skills真正闪耀的地方。

#### 可重复的工作流程

在非确定性系统中，由于我们并不总是知道模型的输出会是什么，因此很难找到可重复产生相同输出的方法。Skills允许我们提供具有清晰步骤或指令的可重复工作流程，使代理能够执行我们能够更准确地预测的任务。

#### 新能力引入

Skills还引入了新能力的概念——代理本身不知道如何开箱即做的功能，甚至Claude不知道如何操作的数据。当我们引入这些新能力时，就用最少的额外上下文为代理释放了整个生态系统和新功能。

例如生成演示文稿、Excel电子表格、PDF报告、必要时执行脚本等——这些都是代理Skills可以发挥作用的领域。

### 四、Skills的可移植性和可组合性

#### 跨环境使用

虽然到目前为止我们在Claude AI中看到了Skills，但Skills可以以完全相同的格式使用，不仅在Claude Code、Agent SDK和API中，而且由于Agent Skills是开放标准，用户可以在越来越多的代理产品中使用。

这意味着用户可以创建一个Skills，并在许多不同环境中使用、分享和扩展。

#### 可组合性

Skills具有可组合性——用户可以将自定义Skills（如营销活动分析）与内置Skills（如创建PPT、PDF或Excel电子表格）结合使用。用户不仅可以同时使用多个Skills，还可以将它们组合起来构建复杂且可预测的工作流程。

通过引用必要的Skills和必要的步骤，用户可以在非确定性系统中创建可预测的输出。

### 五、Progressive Disclosure（渐进式披露）机制

#### 上下文窗口保护

Skills在底层可以包含相当多的信息，有额外的Markdown文件示例，甚至有可执行脚本的示例。用户可以在系统中拥有数百个Skills。

为保护上下文窗口，Skills采用渐进式披露机制。渐进式披露的理念是只加载必要的数据，避免污染上下文。将上下文窗口视为公共资源：添加到上下文窗口的数据越多，消耗的令牌就越多，上下文窗口填满的速度就越快，上下文降解或错误响应的可能性也可能增加。

#### 分阶段加载流程

为避免用可能不需要的数据污染上下文窗口，Skills引入渐进式披露机制。当Skills从文件系统加载时，唯一添加到上下文窗口的数据是Skill的名称和描述——这对于Claude或任何其他系统知道Skill是什么以及如何触发它至关重要。

一旦Skill被触发，底层的SKILL.md就会被加载——这是将数据加载到上下文的下一阶段。根据需要，如果需要加载和执行额外文件或脚本，这些将逐步加载。这些额外资源可以按需加载，如果有需要加载的脚本，这些脚本会在上下文窗口外加载和执行，以避免用不必要的额外令牌污染上下文。

#### 智能资源利用

通过使用bash和文件系统等工具，Claude可以只加载必要的信息，只执行必要的脚本和文件读取，并有意识地只向上下文窗口添加必要的内容。

## 关键引述/重要观点

> "Similar to the model context protocol, skills themselves are an open standard that can be used across many different AI applications."

> "While skills were something originally created at Anthropic. Skills themselves are now an open standard with a specific specification that is used across many different platforms, including Codex, Gemini CLI, Claude Code, Open Code, and much more."

> "It's important to note that skills are not just text files that reference other text files, but text files that can reference scripts, what they do, and when they need to be executed."

> "Where skills really shine are places where Claude might not know exactly how you or your company operates."

> "These simpler agents are easier to evaluate, understand, and scale. But what these agents lacked was the underlying context and domain expertise to do the job reliably."

> "We want finance agents to perform financial analysis in a particular fashion. We want research agents to have the domain expertise necessary to research the way that we want."

> "These skills provide us the procedural knowledge and the user-specific context that they can load on demand."

> "What skills allow us to do is provide a repeatable workflow with very articulate steps or instructions that allow the agent to perform a task that we can start to predict with more accuracy."

> "When we bring in these new capabilities, we unleash an entire ecosystem and new functionality for our agents with minimal additional context."

> "We like to think of the context window as a public good. The more data that we add to the context window, the more tokens we consume, the faster our context window fills up, and the likelihood of context degradation or incorrect responses potentially increases."

> "The idea of Progressive Disclosure is to only load the data necessary and avoid polluting the context."

> "By using tools like bash and a file system, Claude can load only the information that's necessary, execute only scripts and reading of files that is necessary, and intentionally only add what is necessary to the context window."
