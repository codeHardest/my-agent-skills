本视频深入讲解了如何在Claude Code中使用Skills进行代码生成、审查和测试。演示了通过CLAUDE.md文件设置项目上下文，创建Skills定义工作流程规范，以及创建配备Skills的Sub-Agents来实现代码审查和测试生成的高效协作模式。

## 核心要点
1. **CLAUDE.md文件**：用于指定项目通用指令、技术栈和Claude在每个对话中需要了解的信息，位于项目根目录，始终存在于上下文中
2. **Skills的创建与管理**：Skills定义在.claude/skills文件夹中，可在项目级别和用户级别创建，包含名称、描述和工作流程规范

4. **generating-cli-tests技能**：用于生成pytest测试，包含fixtures定义、测试结构模式、边缘用例覆盖清单和测试运行命令
5. **reviewing-cli-command技能**：用于代码审查，确保遵循最佳实践，检查结构、装饰器、注册方式、类型注解等
6. **Sub-Agents的创建**：使用/agents命令创建，需要手动配置名称、描述、工具和技能，子代理不继承父代理的技能
7. **Sub-Agents技能加载机制**：当子代理被调度时，整个SKILL.md文件会被预加载，而不是渐进式披露
8. **多技能协作**：主代理专注于开发，子代理分别负责审查和测试生成，提高上下文效率
9. **项目架构模式**：命令行任务管理应用使用Python + Typer + dataclasses + Rich，数据存储在JSON文件，使用uv管理依赖
10. **技能适用性**：Skills可以跨不同CLI框架和项目使用，只需适配特定的模式规范

## 详细内容（保留所有原始信息）

### CLAUDE.md文件的作用与结构
CLAUDE.md文件是Claude Code中的核心项目配置文件，通过/init命令或手动创建。该文件具有以下特点：
- 始终存在于项目上下文中，每个对话都可用
- 用于指定代码库的一般说明、技术栈和必要信息
- 帮助Claude了解文件位置和最佳信息结构方式
- 本项目示例中指定了：命令行任务管理应用使用Python、Typer框架、dataclasses、Rich库、JSON文件持久化、uv依赖管理
- 架构遵循特定模式：入口点+独立命令文件+models.py数据类+storage.py存储逻辑+display.py显示逻辑+常量+测试

### Skills的定义与结构
Skills定义在.claude/skills文件夹中，包含以下核心要素：
- **名称和描述**：标识技能用途
- **触发条件**：明确何时使用该技能（如"当用户要求编写测试时"）
- **工作流程规范**：详细定义执行任务的具体步骤
- **模式与约定**：具体的代码风格和实现模式，如类型注解方式、参数装饰器等
- **示例**：提供正面和负面示例帮助Claude理解预期行为
- **检查清单**：确保所有最佳实践都被遵循

Skills可在两个级别创建：
1. **项目级别**：在.claude/skills文件夹中
2. **用户级别**：在用户主目录中

### 三个核心Skills详解

**1. adding-cli-command技能**
该技能定义了在Typer框架中添加CLI命令的最佳实践：
- 文件创建位置规范（src/task/commands/目录）
- 命令注册到__init__.py文件
- 类型注解使用Annotated类型（更现代的方式）
- 参数装饰器模式规范
- 显示方法调用（display.success、display.info等）
- 标志位（flags）的简写和完整写法
- 帮助文本要求
- 返回值规范
- docstring要求
- 退出码处理
- 破坏性命令确认机制
- 命令组注册方式
- 适用于跨平台CLI应用开发

**2. generating-cli-tests技能**
用于生成pytest测试覆盖Typer命令：
- 触发条件："当用户要求编写测试时"或"添加测试覆盖"
- fixtures定义：临时存储配置、示例数据、测试基础设施
- 测试结构模式：arrange（设置）→ act（执行）→ assert（断言）
- 测试场景覆盖：按命令类型分类（读取、添加等）
- 边缘用例清单：无效输入、状态处理、确认提示、不存在的资源处理
- 测试运行命令：uv run pytest、verbose模式、特定文件运行
- 提供具体测试代码示例

**3. reviewing-cli-command技能**
用于代码审查和质量验证：
- 检查内容：文件位置、装饰器使用、注册方式、类型注解、参数选项、标志位
- 提供正面和负面示例
- 错误处理和输出检查
- 输出格式：包含检查清单执行结果、摘要、建议修复
- 可作为代码审查流程的一部分
- 确保生产级功能的质量

### Sub-Agents的创建与配置
Sub-Agents通过/agents命令创建，支持手动配置：

**创建流程：**
1. 输入/agents命令创建新代理
2. 选择手动配置而非AI生成
3. 配置代理名称（如code-reviewer、test-generator-runner）
4. 添加描述说明触发条件
5. 配置工具权限（按需选择Bash、Glob、Grep、Read、Edit、Write等）
6. 选择模型继承设置
7. 选择视觉颜色
8. 保存代理
9. 关联所需Skills

**code-reviewer代理配置：**
- 唯一标识符：code-reviewer
- 工具：Bash、Glob、Grep、Read
- 颜色：紫色
- 技能：reviewing-cli-command
- 职责：代码质量、安全、惯例检查

**test-generator-runner代理配置：**
- 唯一标识符：test-generator-runner
- 工具：Bash、Glob、Grep、Read、Edit、Write
- 颜色：黄色
- 技能：generating-cli-tests
- 职责：测试发现、生成、运行

**关键特性：**
- Sub-Agents不继承父代理的技能，需要显式指定
- Skills在代理调度时被完全预加载（整个SKILL.md）
- 不执行渐进式披露或额外文件读取
- 主代理可专注于开发，子代理并行处理审查和测试
- 提供上下文高效的协作模式

### 实际应用演示

**应用背景：**
- CLI任务管理应用，具有add、done、list命令
- 使用Python + Typer + Rich + JSON持久化
- 架构：models.py（数据模型）、storage.py（存储逻辑）、display.py（显示）、各命令独立文件

**编辑命令实现：**
1. 使用adding-cli-command技能创建edit.py命令
2. Claude Code自动提示使用该技能
3. 读取现有文件理解规范
4. 创建edit.py并注册到__init__.py
5. 测试命令功能

**Sub-Agent协作流程：**
1. 调度code-reviewer代理审查edit.py
2. 发现警告并获取修复建议
3. 调度test-generator-runner生成测试
4. 主代理实现建议修复
5. 运行测试验证（uv run pytest -v）
6. 所有测试通过

**clear.py修复案例：**
- 初始状态：未遵循最佳实践，存在6个关键问题和4个警告
- 问题类型：使用了不正确的显示方法、标志位格式错误、退出码不规范、缺少确认机制
- 修复流程：
  1. code-reviewer识别问题
  2. test-generator-runner生成测试
  3. 主代理应用修复
  4. 验证所有问题解决
  5. 测试覆盖确认
