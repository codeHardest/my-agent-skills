## 内容概述
本课程作为最后一课，演示了如何使用Claude Agent SDK构建一个研究代理，该代理能够利用技能（Skill）基于开源工具的文档、GitHub仓库和网络搜索创建学习指南。研究代理包含一个主代理协调器和三个子代理（文档研究人员、仓库分析人员、网络研究人员），它们并行工作以收集信息，最终整合成结构化的学习路径，并通过MCP服务器将结果写入Notion等外部平台。

## 核心要点

1. **Claude Agent SDK定义**：Claude Agent SDK是一种编程方式，用于构建使用与Claude Code相同内部架构的代理应用程序。

2. **主代理架构**：主代理作为编排器，有权访问三个可用的子代理，分别负责文档查找、仓库结构分析和网络内容研究。

3. **子代理工具配置**：
   - 文档研究人员：使用WebSearch和WebFetch工具
   - 仓库分析人员：使用WebSearch、Bash命令（git clone/git命令）和文件读取工具
   - 网络研究人员：使用WebSearch和WebFetch工具

4. **技能（Skill）机制**：技能用于指导主代理的工作流程，定义研究方法、需要提取和综合的内容，使用.md文件格式存储在.claude/skills目录中。

5. **渐进式学习结构**：采用渐进式披露（progressive disclosure）方法，将学习内容分为多个级别，从概述和动机、核心概念、实用模式到进阶方向。

6. **MCP服务器集成**：通过MCP服务器连接外部服务（如Notion），使代理能够将输出写入集中式平台以便团队共享。

7. **项目初始化**：使用`uv init`初始化项目，添加依赖（claude-agent-sdk、python-dotenv、asyncio），创建agent.py文件。

8. **allowed_tools配置**：必须明确列出主代理和子代理需要使用的所有工具，包括Write、Bash、WebSearch、WebFetch等读写工具。

9. **setting_sources配置**：通过setting_sources参数指定技能文件的位置，包括用户目录和项目目录。

10. **并行执行能力**：子代理可以并行启动和执行，同时从不同数据源获取信息，提高研究效率。

11. **目录结构生成**：代理能够自动创建README文件、资源文件夹、学习路径文档和代码示例文件夹。

12. **Notion集成示例**：演示如何使用Notion MCP服务器将resources.md文件内容写入Notion页面，使用批量工具添加内容。

13. **安全考虑**：当前配置允许write和bash命令无用户确认执行，未来需要构建类似Claude Code的确认界面。

14. **实际演示案例**：使用MinerU（PDF提取开源库）作为示例，展示从研究到创建完整学习指南的完整流程。

15. **可扩展性**：可以添加额外的技能以支持更复杂的工作流，或添加更多子代理执行各种任务。

## 详细内容（保留所有原始信息）

### Claude Agent SDK概述

Claude Agent SDK是一种编程方式，用于构建自己的代理应用程序，这些应用程序使用与Claude Code相同的内部架构。课程将构建一个通用研究代理，主代理能够从多个来源研究信息并综合摘要，同时调度三个不同的子代理：
- 分析文档的文档研究人员
- 分析和下载仓库的仓库分析人员
- 通过网络搜索研究信息的网络研究人员

### 主代理提示词设计

主代理提示词作为编排器，明确指定了以下内容：
- 三个可用子代理及其能力描述
- 如何处理技能提供的指令
- 如果技能匹配用户请求，需要精确遵循技能的指令
- 子代理生成和结果综合的高级委托指南

### 子代理提示词设计

**文档研究人员代理**：
- 工具：WebSearch、WebFetch
- 输入格式：特定输入格式指南
- 输出格式：返回特定格式的发现

**仓库分析人员代理**：
- 工具：WebSearch（查找仓库）、Bash命令（git clone和git操作）、文件读取和查找工具
- 提供过程、输入格式、指南和输出格式

**网络研究人员代理**：
- 工具：WebSearch、WebFetch
- 功能：搜索相关内容、接收提取指令
- 输出格式：如果未指定，使用默认结构

### learning-a-tool技能设计

技能名称：learning-a-tool

目的：引导主编排器创建编程工具的学习路径

工作流程：
1. 研究阶段 - 为官方文档子代理指定具体查找内容
2. 仓库分析阶段 - 类似的研究方法
3. 网络研究阶段 - 类似的搜索策略

数据组织：
- 使用渐进式披露方法，加载另一个markdown文件作为真相来源
- 创建多个级别：概述和动机、安装、核心概念、实用模式、进阶方向

目标：获取包含概述、资源、路径和代码示例的学习环境，将所有子代理的研究结果整合成特定输出格式。

### 代码实现步骤

**步骤1：项目初始化和依赖安装**
- 使用`uv init`初始化项目
- 添加依赖：claude-agent-sdk、python-dotenv、asyncio

**步骤2：创建agent.py基础文件**
- 导入asyncio、dotenv、display_message工具
- 设置基础Claude代理配置（system_prompt、allowed_tools）
- 创建简单对话循环

**步骤3：添加工具和MCP服务器**
- 添加Write、Bash、WebSearch、WebFetch工具到allowed_tools
- 配置MCP服务器（notion服务器）
- 从.env文件加载Notion token
- 添加Notion MCP工具到allowed_tools

**步骤4：加载子代理定义**
- 导入AgentDefinition类
- 创建子代理字典，包含每个子代理的描述、提示词和工具
- 更新system_prompt为主代理提示词
- 添加agents参数传递代理定义字典
- 确保allowed_tools包含所有子代理需要的工具
- 添加Task工具以支持子代理调度

**步骤5：添加技能支持**
- 添加Skill工具
- 配置setting_sources参数指定技能位置
- 技能文件应命名为SKILL.md，存放在.claude/skills目录

### 实际演示：MinerU学习指南创建

**演示场景**：
- 目标：学习MinerU（开源PDF提取库）

**计划阶段**：
- 调用learning-a-tool技能

- 显示根据技能要求的结构
- 显示预期输出

**执行阶段**：
- 并行启动docs_researcher、repo_analyzer、web_researcher子代理
- 文档研究人员访问官方文档

- 网络研究人员在YouTube和其他平台搜索教程

**结果生成**：
- 创建完整学习指南目录结构
- README文件：学习路径、使用方法、时间估计

- 学习路径文件：概述与动机、问题背景、图书馆起源、核心功能、代码示例

**Notion集成**：
- 将resources.md内容写入Notion子页面
- 使用Notion MCP服务器批量写入工具
- 将markdown内容转换为Notion blocks格式
- 动态更新Notion页面内容
