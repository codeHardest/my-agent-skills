### 技能创建方法论

#### 基础与渐进式披露原则
When creating a skill, start with basic markdown instructions, then expand later following progressive disclosure.

技能创建应该采用"从小到大、从简单到复杂"的策略。最初阶段，使用基础的markdown格式编写说明文档，确保技能的核心功能清晰可用。随着需求的发展和场景的丰富，再逐步扩展技能的细节和功能。这种方法有助于保持技能的可维护性和可扩展性。

#### 监控与迭代机制
Monitor how your agent uses your skill in real scenarios and iterate based on observations.

技能的优化是一个持续的过程。在真实的应用场景中观察agent如何使用技能，收集实际使用数据和问题反馈，然后基于这些观察进行迭代改进。这种数据驱动的迭代方法确保技能能够不断适应实际需求。

#### 描述的完整性要求
Make sure the description contains enough detail for your agent to know when to use it.

技能的描述部分是确保agent能够正确使用技能的关键。描述需要包含足够的上下文信息，包括技能的使用场景、触发条件、功能范围等，使agent能够准确判断在何时何地应该调用该技能。
