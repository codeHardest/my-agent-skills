## 详细内容（保留所有原始信息）

### 1. Claude API与Claude AI/Claude Desktop的差异

在使用Claude API时，有两个关键点需要注意：

**第一个重要事项**：在Claude AI和Claude Desktop中创建的Skills不会自动共享到Claude API或Claude Code中。这意味着用户需要在每个环境中独立创建和管理自己的Skills。

**第二个重要事项**：为了让Skills正常工作，需要具备以下能力：
- 执行代码
- 创建和编辑文档、演示文稿、PDF、数据报告
- 文件系统操作

这些能力在Claude AI和Claude Desktop中是默认配置的，但在使用Claude API时需要手动启用。

在Claude Desktop或Claude AI中，用户可以在设置中的"capabilities"部分找到"code execution and file creation"选项。这是默认启用的设置，允许Claude执行代码、创建文档、电子表格、演示文稿等。这本质上为Claude AI和Claude Desktop提供了一个执行代码的计算机或虚拟机。

### 2. 代码执行工具（Code Execution Tool）详解

当使用Claude Code和Claude Agent时，用户可以直接访问文件系统。但使用Claude API时则不同，需要一个容器来执行代码和一个文件系统来工作。

代码执行工具的功能包括：
- 允许Claude运行Bash或shell命令
- 创建、查看、编辑文件和编写代码
- 所有操作都在沙箱环境中执行

**容器化环境的工作原理**：
- 代码执行工具为应用程序提供了一个独立的专用容器来执行代码
- 当要求Claude创建和执行文件时，这些操作在安全隔离的环境中进行
- 可以向该文件系统中添加目录

**限制条件**：
- RAM、磁盘、CPU资源限制
- **无互联网连接**（这是Messages API特有的限制）
- 预安装库（开箱即用）

**重要说明**：在Claude AI或Claude Desktop中使用代码执行工具时，确实有互联网连接，可以下载和安装包。

### 3. Files API详解

当处理文件时（添加、创建、写入、修改），需要一种机制来存储底层文件。Claude API包含一组称为Files API的接口，用于上传和下载可以在容器内运行和处理的文件。

**典型使用场景**：
用户要求总结输入内容并保存摘要到文本文件 → 上传输入文件 → 发送到容器 → 下载生成的文件

Files API的工作流程：
1. 使用files_from_dir辅助函数上传技能目录
2. 使用必要的beta headers进行请求
3. 获取file_id用于后续下载
4. 通过file_id下载生成的文件

### 4. Skills在API中的使用方式

Skills是Claude API功能的核心组成部分。在Claude AI或Claude Desktop中内置的Skills库，或者使用API可以包含的Skills，都位于容器中由Skills目录驱动的目录中。

**关键要求**：当使用API时，如果想使用Skills，也需要使用代码执行工具。

**Skills的配置方式**：
- Skills设置在名为container的关键字参数中
- 可以传递自定义Skills或内置Skills
- 创建多个版本的Skills时，可以引用特定时间戳或使用最新版本

**Beta Headers的配置**：
使用Sonnet时，需要传递必要的beta headers：
- skills：用于Skills功能
- code_execution：用于代码执行
- files API：用于文件上传（发送文件时必须包含）

### 7. 渐进式披露（Progressive Disclosure）机制

渐进式披露是Skills设计的重要概念：

- 模型首先只读取SKILL.md文件
- 如果需要额外文件，会按需读取
- 不一次性加载所有文件，优化资源使用
- 随着任务进展，逐步使用更多Skill资源
