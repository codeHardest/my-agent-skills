### 课程介绍与讲师介绍

### Skills的定义与核心概念
Skills是包含指令的文件夹，用于扩展代理的能力，为其提供专业化知识。Skills的核心价值在于将特定领域的知识和工作流程封装成可复用的模块，使代理能够更好地处理特定类型的任务。通过这种方式，用户不需要每次都详细解释相同的工作流程，代理可以自动了解如何处理相关任务。

### Skills的开放标准特性
课程强调Skills的一个重要特性：它们现在是开放标准（open standard）。这意味着Skills采用标准化的格式，可以与任何兼容Skills的代理一起工作。这一设计理念使用户能够"一次构建，多处部署"（build your skills once and deploy them across multiple agent products），大大提高了Skills的可移植性和通用性。

### SKILL.md文件结构
每个Skill都应包含一个SKILL.md的Markdown文件，该文件是Skill的核心组成部分。SKILL.md文件包含以下内容：
- **名称（name）**：Skill的标识名称
- **描述（description）**：Skill功能的简要说明
- **主要指令（main instructions）**：定义Skill具体行为的指令

主要指令可以进一步引用其他文件类型，包括：
- 脚本文件（scripts）
- 额外的Markdown文件（additional markdown files）
- 资产文件（assets），如模板（templates）和图片（images）

### 渐进式披露机制
Skills采用"渐进式披露"（progressively disclosed）的工作机制，这是其高效运行的关键：

1. **始终可见**：Skill的名称和描述始终存在于代理的上下文窗口中，确保代理能够识别当前请求是否与特定Skill相关
2. **按需加载**：代理不会立即将所有指令加载到上下文中，只有当用户请求与Skill描述匹配时，才会加载其余指令
3. **动态补充**：此时代理可能会额外加载相关的引用文件和资产文件（如需要）

这种机制确保了上下文窗口的高效利用，避免了不必要的信息过载。

### 基本工具要求
要使用Skills，代理需要具备一套基本工具：
- **文件系统访问**：用于读取和写入文件
- **Bash工具**：用于执行代码命令

这些工具使代理能够执行任何Skill所需的具体命令，实现各种复杂的工作流程。

### Skills与MCP的集成
代理可以将Skills与MCP（Model Context Protocol）结合使用，创建强大的代理工作流程：
- 使用MCP从外部数据源获取数据
- 依靠Skill来确定如何处理这些数据或如何高效地检索数据
- 实现数据获取与数据处理的分离与协作

### Skills与子代理的协作
Skills还可以与子代理配合使用：
- 代理可以将任务委托给具有隔离上下文的子代理
- 子代理本身也可以使用Skills来获取专业化知识
- 这种分层架构使得复杂任务的分解和协作变得更加灵活

### 何时使用Skills
课程最后解答了一个关键问题："如何知道何时使用Skill？"

使用场景：当你有一个反复要求代理执行的工作流时，不需要每次都详细解释相同的工作流程，可以将其打包为一个Skill。这样代理就会自动知道如何处理该任务，无需重复说明。这一概念将在第一节课程中详细展开讲解。

For example, it can use MCP to get data from external sources, then rely on a skill to know what to do with that data or how to retrieve it efficiently. It can also delegate tasks to a sub-agent with isolated context, which can itself use skills for specialized knowledge. And finally, we'll build a research agent with the Claude agent SDK that uses a skill to combine research results. I'd like to thank Hawraa Salami from DeepLearning.AI who contributed to this course. So, how do you know when to use a skill? Let's say you have a workflow that you repeatedly ask your agent to implement.
