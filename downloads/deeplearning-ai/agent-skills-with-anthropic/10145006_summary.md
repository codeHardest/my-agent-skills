## 详细内容（保留所有原始信息）

### 一、技能的必需结构与YAML Frontmatter

每一个创建的技能都包含一个必需的SKILL.md文件，该文件顶部包含YAML Frontmatter，其中name和description是必须字段。SKILL.md的主体部分包含技能的实际内容，以及对脚本、附加文本文件或必要资产的引用——这些资源仅在需要时才被加载。

名称和描述的设计被认为是关键任务，因为它们不仅决定了Claude如何理解技能的功能，还决定了系统何时应该触发使用该技能。名称有最大字符数限制，描述同样有最大字符数限制。名称必须包含小写字母、数字和连字符，通常应采用"动词+ing"的命名形式（如"generating-practice-questions"）。描述部分不仅需要描述技能的功能，还需要描述使用场景，如果存在能够触发代理激活该技能的特定关键词，务必将这些关键词纳入描述中。

除了必需的字段外，Agent Skills规范还允许可选字段，包括license、compatibility以及元数据中的任意键值对。需要注意的是，尽管存在技能规范标准，但可能遇到一些技能（包括Anthropic构建的和其他来源的）并不完全遵循该规范。技能和技能规范都在积极开发中，因为团队需要与许多不同的模型提供商和代理工具生态系统合作。

### 二、技能主体内容的最佳实践

从YAML Frontmatter进入技能主体后，对于格式没有底层限制。然而，当考虑构建可预测的工作流时，必须确保提供分步骤的指令。正如在skill-creator等其他技能中所见，指定边界情况、分步骤说明以及明确说明跳过某步骤的原因都非常重要。

一般而言，将技能控制在500行以内是最佳实践，因为当需要时可以引用外部文件、资产和脚本。保持清晰和简洁是有价值的，即使在Windows上使用正斜杠也至关重要。必须确保技能能够在许多不同的环境中工作。

创建技能时需要考虑给予技能多大的自由度：应该允许通用的方法和方向，还是应该专注于特定的序列？对于遵循最佳实践的场景，可能需要低自由度；但对于创意输出、多种颜色、多种样式、多种字体的场景，可以允许高自由度。

当开始考虑更复杂的工作流和多技能场景时，将任务分解为顺序步骤总是比拥有一个试图完成所有功能的单一庞大技能更有价值。这些系统可以处理100+个技能，确保它们命名适当、不混淆、遵循可预测的模式非常重要。

### 三、可选目录结构详解

在规范中，存在可选目录结构的空间。正如在许多不同技能中所见，存在用于scripts、references和assets的子文件夹。

**Scripts目录**：包含任何需要读取和执行的代码类型，同时需要确保有错误处理和清晰的文档。

**References目录**：包含附加文档或参考文件，通常指示技能在文档较长时阅读整个参考文件是有价值的。

**Assets目录**：包含输出模板、图片、徽标、数据文件、模式等资产。需要注意的是，这些目录——scripts、references和assets——遵循Agent Skills的标准。但可能遇到许多不一定遵循该标准的技能。该标准正在快速演变，技能也在快速演变。因此，预计未来创建的技能将遵循此标准，但可能会遇到具有不同文件夹名称和不同约定的技能。

### 四、示例技能一：生成练习题（Generating Practice Questions）

第一个自定义技能是用于从讲义生成教育练习题的技能。描述内容为：从讲义生成教育练习题以测试理解能力。假设用户是教师或讲师，想要提供特定的输入和输出格式，生成全面的问题来测试理解能力。

**技能结构解析**：
- 首先定义支持的输入格式，指定要使用的特定库和要提取的文本内容
- 然后定义问题结构，指定希望按特定顺序生成问题：从是非题开始，一直到现实应用题
- 每个问题类型都有对应的子指南
- 该技能不超过500行代码，如果需要增长，可以包含底层参考文件

**输出格式处理**：
- 输出格式取决于用户请求
- 不直接提供每种输出格式的示例，而是引用assets文件夹中的模板
- 对于LaTeX或Markdown格式，精确指定输出格式
- 例如Markdown格式的真假题样式，LaTeX格式的问题示例样式

**渐进式披露应用**：
- 这些模板文件仅在需要时加载
- 可以通过仅加载特定数据格式的文件来提高token和上下文窗口效率
- 如有外部资源需求（如领域特定示例），可以链接到references文件夹中的内容
- 通过仅加载绝对必要的内容并仅在需要时引用外部文件来实现渐进式披露

### 五、示例技能二：时间序列分析（Analyzing Time Series）

第二个技能用于分析时间序列数据特征。场景是提供CSV文件，需要在预测前理解数据的多个特征。该技能具有非常特定的确定性工作流，利用多个Python脚本来执行操作。

**Python脚本组件**：
- **viz.py（可视化脚本）**：用于绘制时间序列图、直方图、滚动统计、箱线图等
- **autocorr.py（自相关脚本）**：用于绘制自相关图
- **decomp.py（分解脚本）**：用于分解数据
- **diagnose.py（诊断脚本）**：包含分析数据的功能，执行数据质量分析、分布分析、平稳性测试、季节性检测、趋势分析、自相关分析，最终给出变换建议

**工作流设计**：
- 首先明确定义输入格式，指定要查找的内容、列名和数据类型
- 工作流是最重要的部分，需要非常明确地说明要运行的精确脚本
- 可选步骤是生成必要的图表并向用户报告数据
- 从summary.txt中提取数据并呈现相关图表
- 使用interpretation.md文件提供指导

**脚本选项与依赖**：
- 可以根据需要添加额外的标志
- 精确指定输出文件树结构，包括文本文件、图片等
- 确保数据输入、操作和最终输出的高度可预测性
- 列出外部参考资料
- 明确标注Python库依赖，确保安装以便脚本正确运行

### 六、使用Claude Code进行技能评估

使用skill-creator技能评估创建的两个技能是否符合最佳实践。Claude Code不像Claude AI那样内置包含skill-creator等技能，因此需要通过marketplace安装。

**安装流程**：
- 访问Marketplaces
- 添加anthropic/skills的marketplace，该仓库包含两个集合：
  - document-skills：处理Excel、PowerPoint、Word和PDF
  - example-skills：包括skill-creater等示例技能
- 在项目范围内安装
- 需要重启Claude Code
- 验证settings.json中的enabledPlugins包含这些技能

**并行评估方法**：
- 使用两个子代理并行评估两个技能
- 每个子代理评估一个自定义技能
- 系统会自动加载skill-creator技能
- 读取必要文件并派发子代理检查最佳实践
- 找到正确的技能：生成练习题和分析时间序列

**评估结果**：
- **生成练习题技能**：9/10分，可在简洁性方面改进，获得了一些良好建议
- **分析时间序列技能**：表现更好，在避免重复、Frontmatter质量和简洁性方面表现出色

### 七、技能的单元测试方法

将技能通过skill-creator进行了最佳实践分析，但如何确保技能按预期工作是下一步需要考虑的问题。可以构建类似软件单元测试的测试框架来验证技能。

**生成练习题技能的测试**：
- 从多个不同查询开始测试：生成问题并保存为markdown文件、LaTeX文件、PDF
- 验证传入正确格式的正确文件
- 确保预期行为符合要求：使用正确的PDF输入库、提取指定的学习目标、生成不同类型的问题并遵循指南
- 验证使用正确的输出结构、正确的输出模板
- 确保LaTeX等特定数据格式成功编译
- 验证问题生成到正确文件和正确格式
- 在此过程中收集人工反馈
- 在计划使用的所有不同模型上进行测试

**时间序列分析技能的测试**：
- 假设Python脚本已通过传统软件单元测试验证
- 测试工作流的正确顺序和适当的输入输出
- 测试场景：分析和生成时间序列数据的图表
- 传入可能的CSV文件
- 确保viz.py和diagnose.py正确运行
- 确保工作流中的所有步骤按正确顺序执行
- 如果请求图表，确保包含该可选步骤
- 返回摘要并解释发现
- 创建文件夹放置所有必要文件到正确位置
- 获取人工反馈并在所有使用的模型上测试
